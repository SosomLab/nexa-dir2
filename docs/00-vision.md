# 00 · 비전 & 목표

## 1. 한 줄 정의

> **Nexa Dir 2** — Nexa Dir(원본)의 기능 경험을 **포터블 단일 exe · 초저메모리 · 순수 네이티브**로 다시 짓는 Windows 파일 탐색기.

## 2. 문제 정의 (Why)

원본 [Nexa Dir](https://github.com/SosomLab/nexa-dir)(Rust 코어 + WinUI 3/C#)은 M1 후반까지 구현·검증됐지만,
스택 구조상 다음 한계가 실측으로 확인되었다.

- **메모리**: 유휴 RSS **60MB+** — .NET CLR(~25–35MB) + WinAppSDK/WinUI 컴포지션(~20–30MB)이 하한을 결정.
  경쟁 지표인 Double Commander는 **20MB 미만**(Win32/LCL·비관리 언어).
- **배포**: WinUI 3의 XAML/PRI 리소스 해석이 self-contained 게시에서 반복적으로 결함
  (원본 BUG-010: loose XAML→x:Bind 붕괴, WinAppSDK 1.6 힙 손상). **단일 exe는 미검증·고위험**.
- **크기·의존**: self-contained ≈64MB, WinAppSDK 런타임·수십 개 dll 동반 — "포터블 단일 exe·dll 최소화" 목표와 상충.

원인은 기능이 아니라 **관리(managed) 런타임 + UI 프레임워크 계층**이다. 따라서 이 저장소는
**unmanaged 올 러스트 스택**으로 동일 기능을 재구축한다. 원본 저장소는 설계·기능 명세·실측 교훈의 **원천(SSOT)** 으로 계속 참조한다.

## 3. 목표 (Goals) — 예산(Budget)이 곧 정체성

| # | 목표 | 성공 지표 (북극성) |
| --- | --- | --- |
| G1 | 초저메모리 상주 | 유휴 RSS **≤ 30MB**(목표 20MB급, Double Commander 동급) |
| G2 | 진짜 포터블 | **단일 exe ≤ 10MB**, 외부 파일 0(영속물은 exe 옆 `data\` 선택), 설치 불요 |
| G3 | 의존 최소 | 임포트 DLL = **OS 인박스만**(kernel32/user32/gdi32/shell32/ole32 등), 재배포 런타임 0 |
| G4 | 즉시 반응 | 콜드 스타트 **< 200ms** · 100k 항목 폴더 첫 렌더 **< 150ms** · 스크롤 60fps (원본 G1 계승) |
| G5 | 기능 패리티 | 원본 M1 기능군(플래그십 트리·듀얼/탭·파일 조작·터미널 등)을 순차 재현 |

## 4. 비목표 (Non-Goals, 초기)

- 원본의 대체/폐기 — 두 에디션은 병행 가능(원본=설치형·WinUI, 본 저장소=포터블·네이티브). 수렴 여부는 M2+ 이후 판단.
- .NET 미리보기 플러그인 SDK(`Nexa.Plugins`) 호환 — 관리 런타임 탑재가 예산(G1~G3)과 충돌. 내장 미리보기로 대체(DR-7).
- macOS/Linux UI — 코어는 이식 가능하게 유지하되 1차 타깃은 **Windows**(원본과 동일).
- MSIX/스토어 배포 — 단일 exe 채널에 집중(서명은 후속).

## 5. 타깃 사용자

- **USB/현장 작업자·관리자**: 설치 권한 없는 PC에서 exe 하나로 즉시 사용.
- **파워 유저/개발자**: 상시 상주시켜도 부담 없는 메모리 풋프린트 + 키보드 중심 워크플로우.
- 원본 Nexa Dir 사용자 중 "가볍고 휴대 가능한" 에디션 수요층.

## 6. 경쟁 제품 분석 (포터블 관점)

| 제품 | 스택 | 유휴 RSS | 배포 | 우리의 차별점 |
| --- | --- | --- | --- | --- |
| Double Commander | Free Pascal/LCL(Win32) | <20MB | 포터블 zip | UI 노후 — 우리는 원본의 모던 UX 계승 |
| Total Commander | Delphi(Win32/GDI) | ~15MB | 설치/포터블 | 유료·구식 — 동일 체급 성능 + 차세대 UX |
| Everything (voidtools) | C++/Win32 커스텀 드로잉 | <20MB | 단일 exe 有 | 검색 전용 — 우리는 탐색기 전체 |
| 원본 Nexa Dir | Rust+WinUI 3 | 60MB+ | setup.exe ≈64MB | 기능 원천 — 우리는 예산·포터블 달성판 |

**차별화 포인트**: *원본의 플래그십 UX(인라인 트리+교차선택)* × *Win32 커스텀 드로잉 체급의 예산* × *단일 exe*.
이 조합을 제공하는 제품은 없다.

## 7. 핵심 차별화 기능 (우선순위)

0. **★ 인라인 폴더 확장 + 폴더 교차 다중 선택** — 원본 제1 차별화 계승(코어 `nexa-tree` 그대로 재사용). → 원본 docs/07
1. **예산 보증**: G1~G4를 마일스톤마다 **게이트로 실측**(원본은 사후 최적화로 60MB에 도달 — 우리는 예산 초과 시 병합 금지).
2. **탭 + 듀얼 패널** · 파일 조작 전체(전송엔진·Undo/Redo·셸 메뉴·DnD).
3. **하단 ConPTY 터미널** · 내장 미리보기.
4. 검색/클라우드/AI — 원본 M3+와 동기(후속).
