## UX QA 5차 (main 직커밋) — 비활성 패널 구분·스플리터 잔존·이름 복사 위치·설정 창 X-8

> **사용자 QA/요청 6건**: ① 좌측 선택 후 우측 클릭 시 양쪽 선택 하이라이트가 동일해
> 활성 패널 식별 불가 → 비활성 쪽 색 구분 ② 듀얼 터미널에서 비활성 패널의 [→](폴더
> 이동) 버튼도 accent 파랑 유지 → 비활성 표시 ③ 스플리터 드래그 후 마우스를 떼어도
> 파란 강조가 남음 ④ Copy as name을 Copy as path 바로 밑으로 ⑤ 설정 창을 VS Code
> 방식으로 ⑥ 설정 창 크기 조절.

- **비활성 패널 선택색 구분**(①② — 탐색기 관례): 테마 토큰 `sel_bg_inactive` 신설
  (다크 0x2C313A·라이트 0xE6E9EE — 무채색 블렌드). `VirtualRows`/`InfoDock`에
  `set_focused` 추가, `Panel::set_focused`가 tabbar와 함께 **전 탭 rows·dock**에 전파
  (새 탭·복제 탭은 패널 focused 상속). 도크 스트립 활성 종류 라벨과 [→] 버튼도
  비활성 패널에선 `sel_bg_inactive`(accent 대신). 기동 시 복원된 활성 패널 인덱스
  반영(기존: 무조건 우측만 비활성 처리 — `active=1` 복원 시 불일치).
- **스플리터 강조 잔존 수정**(③): WM_LBUTTONUP에서 `dock_split_drag`(도크 좌/우)·
  `dock_drag`(도크 높이)는 플래그만 끄고 재도장을 안 해 accent가 남음(파일 좌/우만
  invalidate) → 두 경로 모두 해제 시 InvalidateRect.
- **이름 복사 위치**(④ — 원본 PR#10): `CustomItem.after_id` 신설 — 셸 메뉴 병합 시
  지정 ID **바로 아래 InsertMenuW**(제자리 대체된 "경로 복사"[CTX_COPY_PATH] 앵커),
  대상 부재 시 하단 고유 섹션 폴백. Copy as name이 Copy as path 바로 밑에 위치.
- **설정 창 VS Code식 완성 + 리사이즈**(⑤⑥ — X-8, X-7 후속):
  - **즉시 적용**: [저장]/[취소] 제거(pref.save/cancel 키 말소) — 체크박스·테마/언어
    순환=클릭 즉시, 글꼴/크기 EDIT=**포커스 이탈(EN_KILLFOCUS) 시** 적용(키 입력마다
    DWrite 백엔드 재생성 방지). 통지 = `WM_APP_PREFS_APPLY`(0x8006) **동기 SendMessage**
    (포인터는 통지 동안만 유효 — 수신 즉시 복사). 호스트 적용 로직을 `apply_prefs`로
    분리(실시간·닫기 공용, 멱등 — 테마는 무변경 시 생략). 닫기=X, 닫기 시 최종 값
    한 번 더 적용·영속(미이탈 편집 값 수거).
  - **리사이즈**: WS_THICKFRAME+WS_MAXIMIZEBOX(`PREFS_STYLE`), WM_SIZE에서 검색창 폭·
    편집 컨트롤 폭(잔여 폭) 재배치, WM_GETMINMAXINFO 최소 크기=기본 클라이언트 크기.
