## 휴지통 삭제 지연 — 원인 판단 + 낙관적 숨김 (사용자 QA 연속)

- **사용자 판단 확인**: "휴지통 관련 기능 호출이 오래 걸리는 게 근본 원인" —
  **동의**. 근거: ① Shift+Del(완전 삭제 = `std::fs`)는 즉시 처리(사용자
  실측) ② 워커 이관 후에도 파일이 실제로 사라지기까지 수 초 = API 자체
  소요 ③ `SHFileOperationW`(휴지통 이동)는 파일 크기와 무관한 셸 고정
  비용 — $Recycle.Bin `$I`(메타)/`$R`(본체) 생성·볼륨별 휴지통 설정/용량
  확인·SHChangeNotify 브로드캐스트·셸 확장/AV 후킹. 환경 요인(휴지통에
  항목 다수·OneDrive 폴더·실시간 검사)이 크면 3~4초도 정상 범위.
- **우리 코드로 API를 빠르게 만들 수는 없음**(대안 IFileOperation도 동일
  셸 경로) → **탐색기 동일 UX = 낙관적 숨김** 채택: DEL 즉시 대상 행을
  화면에서 제거하고 워커가 뒤에서 휴지통 이동을 마친다.
  - [nexa-tree `remove_paths`](../../crates/nexa-tree/src/lib.rs) 신설 —
    경로(+펼친 하위) 노드를 roots/visible/children/선택에서 제외(arena
    유지 = 잔여 id 안정·FS 무변). 테스트 1건(파일/폴더 하위/무매치).
  - [panel `hide_paths`](../../crates/nexa-app/src/panel.rs) → do_delete가
    양 패널에 즉시 적용. **실패 시 완료 재로드가 원복**(정직 복구).
  - 워커에 COM STA 초기화 추가(셸 확장 대비 — SHFileOperation 권고).
- 진단 안내(환경 측): 휴지통 비우기 후 재시도·대상 폴더가 OneDrive 동기화
  경로인지·셸 확장(TortoiseGit 등)/실시간 AV 개입 여부 확인.

## X-30 전송 진행 UX 개편 (사용자 요청 5건 — `feat/transfer-progress-ux`)

사용자 요청: ① 진행 창 시간을 설정 화면에서 직접 변경(+0=창 미표시) ② "1 B"
용량 오표기 확인 ③ 전송 미완료 대상 실행/이동 제한 ④ 용량 단위 적응 +
진행/전체·파일 수 표시 ⑤ 전체 바를 파일별 비율로 구분해 어느 파일이
완료/진행 중인지 보이게.

- **"1 B / 1 B" 오표기 원인**([dialog.rs](../../crates/nexa-app/src/dialog.rs)
  `set_done`): 완료 표시가 바 100%를 만들려고 `done=1/total=1`로 **실제
  바이트를 덮어써** 완료 순간 "1 B / 1 B (100%)"가 됨(진행 중에는 정상 —
  사용자 스크린샷 재현 일치). 수정 = 완료 시 실제 값 유지(취소·실패 시 부분
  진행도 정직 표기). 0바이트 전송(빈 파일)은 완료 단계에서만 100% 간주.
- **nexa-ops 이벤트 프로토콜**([lib.rs](../../crates/nexa-ops/src/lib.rs)):
  `on_progress(Progress, &Path)` → **`on_event(Event)`** 확장 — `Plan{sizes,
  total_bytes}`(시작 전 1회 — 항목별 크기·총합 선통지) · `ItemStart{index,
  dest}`(실제 대상 경로 — 잠금용) · `Bytes(Progress)` · `ItemEnd{index,
  status(Done/Skipped/Failed)}`. 테스트 2건 추가(계획/시작/종결 검증·항목별
  건너뜀 보고) — 총 35 green.
- **진행 창 개편**(dialog.rs): ① 용량 표기 **B/KB/MB/GB/TB 적응**(`fmt_bytes`
  TB 추가) ② 정보 줄(바 바로 위) = `진행 / 전체 (pct%) · 파일 {cur}/{count}`
  (단일/다중 모두 파일 수 표시 — `ops.fileCount`) ③ **세그먼트 바**
  (`paint_segments`): 항목별 크기 비례 구간을 누적 경계로 분할(오차 비누적),
  상태색 = 완료 accent·진행 accent 부분 채움·건너뜀 회색·실패 적색·대기 배경
  + 구간 경계선(3px 미만 생략). 계획 미도착/512항목 초과 = 단색 폴백.
