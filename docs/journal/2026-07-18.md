## Tab 내비 + 3단 체크·헤더 전체 토글 (`feat/bulk-tab-tristate`)

> **사용자 요청 3건**: ① 입력 후 Tab = 배치 순서 포커스 이동 ② Apply 헤더에
> 체크박스 — 클릭 = 전체 체크/해제, 일부만 체크 = **부분선택(흐릿한 ✓)** +
> NxCheckBox를 **2단(체크/해제)·3단(체크/부분/해제)** 선택 지정으로 보완
> ③ 체크 열 폭 = 필요한 만큼만.

- **Tab 내비**: 모달 루프 3곳(본 다이얼로그·Save·Edit)에 IsDialogMessageW —
  Tab = 생성(배치) 순서. 컨테이너 = **WS_EX_CONTROLPARENT**(NxGroupCard·
  NxTextBox·NxSpin 래퍼 — 탭 착지 = 내부 EDIT, 래퍼 TABSTOP 제거) ·
  방향키 소비 컨트롤(콤보/세그/메뉴버튼)에 **WM_GETDLGCODE = WANTARROWS**
  (IsDialogMessage가 화살표를 포커스 이동으로 뺏는 것 방지 — 그리드는 기왕).
- **NxCheckBox 2단/3단**(`CheckMode::Two|Three`): 상태 0/1/2(해제/체크/부분),
  클릭 순환 3단 = 0→1→2→0 · 부분 = accent 박스 + **bg·accent 50% 블렌드 ✓**.
  NXCHK_GET/SETCHECK 값 확장(0/1/2). 갤러리 = 3단 예시로 교체.
- **그리드 헤더 체크**(Mark::Check + 헤더): 컬럼 0 헤더에 동일 체크박스 —
  상태 = 전 행 집계(전체/부분/해제, 부분 = 흐릿한 ✓·해제 = 백지+외곽선[밴드
  위 구별]). 클릭 = 전체 체크↔해제(부분이면 전체 체크), `NXGR_ROW_ALL`(-2)
  통지 → 호스트 전 행 재동기(bulkrename excluded 일괄 갱신).
- 체크 열 폭 = 글꼴 정사각+여백(≈29px — 타이틀 제거, 헤더 체크가 대신).
- 실기: 24→행 해제 23(헤더 흐릿한 ✓)→헤더 클릭 24→재클릭 0 · 갤러리 3단
  순환 1→2→0 캡처. 186 green·clippy 0.
