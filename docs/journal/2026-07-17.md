## ctl AA 렌더링 — GDI+ DrawCtx 백엔드 (`feat/ctl-gdiplus-aa`)

> **사용자 QA·지시**: ① "체크박스·이미지버튼·텍스트박스 색표현이 raster 이미지
> 느낌" ② DrawCtx로 해결 가능한지 질의 → 진단(도형 프리미티브 부재) 후
> ③ **"GDI+를 쓰되 DrawCtx 백엔드로만, ctl에서 직접 호출 금지"** 확정 +
> ④ "텍스트 렌더링에 GDI+는 부적합"(GDI/DW 유지).

- **진단**: GDI 도형(Ellipse/RoundRect/펜)은 AA가 없고, shape 투명에 쓴
  **1비트 창 리전 클립이 곡선 계단의 진범**. 본창이 매끈했던 건 DirectWrite
  텍스트 + 직사각형(fill_rect)뿐이었기 때문.
- **DrawCtx 확장**(nexa-gui): AA 프리미티브 4종 신설 — `fill_ellipse`/
  `fill_round_rect`/`stroke_round_rect`/`polyline`(둥근 캡·조인). 기본 no-op
  (텍스트·테스트 백엔드 비침습).
- **GdipCtx 백엔드**(`ctl::gdipctx` — **코드베이스 유일 GDI+ 접점**):
  GdiplusStartup 1회(OnceLock)·HDC 래핑·SmoothingMode AA·라운드 경로(4호) —
  gdiplus.dll = OS 인박스(B3 게이트 통과·크레이트 0 유지, Cargo feature만 추가).
  래스터라이저 교체(D2D 승격) 시 이 모듈만 바꾼다.
- **규약 개정(사용자 승인)**: ctl "nexa-gui 미참조" → **DrawCtx 트레이트 참조
  허용**(판매 단위 = ctl + 트레이트 + 백엔드). 컨트롤 코드에 GDI+ 심볼 0.
- **적용 4종**: NxCheckBox(박스·✓)·NxIconButton(원판·+/− — **리전 클립 폐기**,
  모서리 = `Style.behind` 신설 필드[부모 배경색]로 칠하고 AA 블렌드)·
  NxTextBox(본체 필+포커스 링 — 리전 폐기)·NxComboBox(필·셰브론·팝업 외곽선·
  hover 필·✓ — 팝업은 화면 위 오버레이라 리전 유지+내부 AA). HDC 혼용 규약:
  **GDI 텍스트 전에 Graphics 해제**(스코프 블록).
- 갤러리: 타이틀 밴드 위 컨트롤(동작 콤보·+/−)에 `behind = sel_bg` 전달 예시.
- 실기 검증: 원판·포커스 링·필·✓·셰브론 캡처 — 계단 제거 확인 · 기능 회귀
  없음(자동 높이 23px·텍스트 왕복·+ 통지/− 무통지·팝업 선택 SEL=3). 186 green.
