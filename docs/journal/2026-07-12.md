## M1-2 ADR-0002 렌더링 확정 (`feat/m1-adr0002-render`)

> GDI vs DirectWrite GDI interop을 **같은 창에서 실측 비교** 후 확정 — [07 ADR-0002](../07-adr-0002-rendering.md) Accepted.

- **DirectWrite interop 백엔드**(`dw.rs`): `IDWriteBitmapRenderTarget`(자체 메모리 DC = 더블 버퍼) +
  `IDWriteTextLayout` → 커스텀 `IDWriteTextRenderer`(`#[implement]` COM 콜백) → `DrawGlyphRun(COLORREF)`.
  배경은 동일 DC에 GDI ETO_OPAQUE 채우기 — `nexa_gui::DrawCtx` 어휘 변경 없이 구현(추상 검증).
  `windows-core` 직접 의존 추가(매크로 전개 경로, DR-8 원장 기록) · feature `Win32_Graphics_DirectWrite`.
- **비교 하네스**(`win.rs`): F2 = 백엔드 전환 · F3 = 200프레임 스크롤 벤치(동기 UpdateWindow) ·
  타이틀바에 평균 페인트 µs 표시 · `NEXA_RENDER=gdi|dw` 강제. 실측은 WScript.Shell SendKeys로 자동화.
- **실측**(릴리스·1200×800·100k 행·Windows 11 실기): 벤치 **GDI 6,072µs vs DW 4,373µs(−28%)** —
  행당 TextLayout 생성 포함(캐시 미적용 상한치)에도 DW가 빠름. 유휴 RSS 13.30 → **17.40MB(+4.1MB)** ·
  exe 0.21 → **0.23MB** — 전 게이트 내.
- **확정**: DirectWrite GDI interop 채택(속도·한글 폴백 일관성·TextLayout 확장성[트리밍·히트테스트 → M1-4~6]).
  기본 백엔드 DW 전환(스모크: RSS 17.07MB). GDI 경로·F2/F3은 육안 비교용 유지 → **M1-3에서 제거** +
  가시 행 레이아웃 캐시 도입 예정. docs/01 §3·docs/10(ADR 색인·crate 원장) 현행화.
