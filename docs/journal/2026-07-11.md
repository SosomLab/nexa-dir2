## M1-1 nexa-gui 크레이트 분리 (`feat/m1-gui`)

> M1 착수. 설계 방침: **nexa-gui는 플랫폼 중립(의존 0)** — 위젯·스크롤·무효화 로직을 맥 `cargo test`로 검증(docs/11 모델 유지).
> 구체 렌더 백엔드는 ADR-0002(M1-2) 확정까지 nexa-app에 둔다(확정 후 이동 여부 재결정).

- **크레이트 신설**(`63ff331`): `geom`(Rect/Point/Size — Win32 클라이언트 좌표 1:1, 반열림 contains·union·intersects) ·
  `theme`(**원본 docs/39 §4 시맨틱 토큰 차용** — `Nexa*Brush` 10종 대응 + 커스텀 드로잉용 `panel_bg_alt`·`text`·`text_dim` 신설,
  다크/라이트 팔레트, 기본=다크 DR-5, 토큰 키는 안정 계약) · `event`(`InputEvent`{Wheel·Key}·`WheelAccum` 분수 노치 누적기 —
  M0-7 로직 이식) · `draw::DrawCtx`(fill_rect + text_opaque — GDI ETO_OPAQUE와 DirectWrite 공통 어휘) ·
  `widget`(`Widget` trait: bounds/set_bounds/on_event/paint + `Invalidations` — 교차 rect union 병합) ·
  `widgets::VirtualRows`(M0-7 스크롤·가시 행·클램프 로직 이식, `RowSource` 추상 — M1-3에서 nexa-tree 평면 스트림 배선 예정).
  **테스트 17 green**(스크롤 클램프·분수 휠·페이지 이동·bounds 축소 재클램프·가시 행만 paint 방문·무효화 병합).
- **nexa-app 재배선**(`80cbbef`): `win.rs` 책임 = 창 수명·더블 버퍼·**WM_* → InputEvent 번역·Invalidations → InvalidateRect 번역**로 축소.
  `gdi.rs` 신설 — `DrawCtx` GDI 구현(ETO_OPAQUE 텍스트 · 브러시 생성 없는 채우기 = 빈 텍스트 ETO_OPAQUE). 하드코딩 다크 팔레트 3색 →
  `Theme::dark()` 토큰. 실기 확인: 기동·휠/키·WM_CLOSE 정상, **exe 0.21MB·기동 RSS 12.4MB**(게이트 내).

## M0-7 GDI 렌더 스파이크 (`feat/m0-render-spike`)

> 렌더 모델(docs/01 §3) 실증: 창 1개·더블 버퍼·**가시 행만** 그리기 — M1 가상 리스트(M1-3)의 전제 검증.

- **M0-7 렌더 스파이크**(`90a9243`): `win.rs` 확장 —
  - **더블 버퍼**: 메모리 DC+비트맵을 창 상태에 캐시, 클라이언트 크기 불일치 시에만 재생성. `WM_ERASEBKGND` → 1(깜빡임 제거), 화면 반영은 `BitBlt` 1회.
  - **가시 행만**: 합성 `TOTAL_ROWS=100_000` 중 `scroll_row`부터 `클라이언트 높이/행 높이`개만 `ExtTextOutW(ETO_OPAQUE)` — 행 배경(교대 음영)+텍스트를 호출 1회로. 프레임 비용은 창 높이에만 비례(100k 체급은 M1-9 게이트와 동일).
  - **스크롤**: 휠(`WHEEL_DELTA` 잔여 누적 — 트랙패드 분수 노치 대응·3행/노치) + 키보드 ↑↓·PgUp/PgDn·Home/End(키보드 우선 DR-5). 클램프 `[0, TOTAL-가시행]`.
  - **DPI**: `WM_DPICHANGED`에서 폰트(Segoe UI 9pt)·행 높이(20px@96dpi) 재계산 + 권고 rect `SetWindowPos`.
  - **수명**: 창 상태(Box)는 `GWLP_USERDATA` — `WM_NCCREATE` 생성, `WM_NCDESTROY`에서 GDI(폰트·브러시·백버퍼) 해제 후 drop. 다크 팔레트는 M2 테마 전까지 하드코딩.
  - windows feature `Win32_UI_Input_KeyboardAndMouse` 추가(VK_*). **검증(맥)**: windows 타깃 `cargo check`·`clippy` green(경고 0) · `cargo test` 26 green · fmt. 실행 확인은 M0-8(Windows 실기)과 병행.
- **권한 병합**(`8c5f7f8`): `Bash(git -C:*)`가 인자 매칭 안 됨 → `Bash(git -C *)` 추가(병합 편집). 사용자 재강조: settings.json **덮어쓰기 절대 금지** — 영구 메모리에도 기록.

> 사용자 요청: "문서 작성 기준·형태는 nexa-dir을 그대로 차용, 개발 방향 설계 + 마일스톤 + 순차 TODO, 이후 첫 개발 착수. 큰 단위=브랜치, 세부 기능=커밋, push는 명시 요청 시."

- **원본 분석**(대화 선행): 원본은 Rust 코어(cdylib, ABI v7) + WinUI 3(C#/.NET8). 실측 한계 — 유휴 RSS 60MB+(CLR+WinUI 하한), self-contained ≈64MB, 단일 exe는 XAML/PRI 결함(BUG-010)으로 고위험. 무게중심이 C#(FileOps·셸·터미널·설정)에 있음을 확인.
- **방향 확정**: unmanaged **올 러스트 + Win32(windows-rs) + 커스텀 드로잉**(GDI→DirectWrite interop). 코어 3크레이트는 rlib 직접 링크(FFI 폐지). 후보 비교(Slint=라이선스 위반·egui=예산/IME·C++=이점 없음·Delphi=자산 폐기)는 [06](../06-adr-0001-stack.md).
