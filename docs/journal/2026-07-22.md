## X-32 클립보드·DnD UX 보완 4종 (사용자 요청 — `feat/clipboard-dnd-ux`)

사용자 요청 4건을 한 브랜치로 묶어 구현(탐색기 관례 정합).

### ① Ctrl+X 잘라내기 항목 흐림 표시

- **요구**: 탐색기처럼 잘라내기된 파일은 목록에서 약간 흐리게.
- [clipboard.rs](../../crates/nexa-app/src/clipboard.rs): `CUT_MARKS`
  thread_local(UI 스레드 전용) — **OS 클립보드의 '잘라내기' 파일 목록 미러**.
  `sync_cut_marks()` = 클립보드가 이동(Preferred DropEffect=MOVE) 파일
  목록이면 그 경로 집합, 그 외(복사/비파일/비움) = 빈 집합. 변경 시 `true`.
- [win.rs](../../crates/nexa-app/src/win.rs): `AddClipboardFormatListener`
  구독(+시작 시 1회 동기·해제는 WM_NCDESTROY) — **WM_CLIPBOARDUPDATE**에서
  `sync_cut_marks()` 후 변경 시 양 패널 목록만 무효화. 별도 상태 배선 없이
  Ctrl+X·컨텍스트 메뉴 잘라내기·**외부 탐색기 잘라내기**·붙여넣기/복사에
  의한 해제까지 전부 이 길목 하나로 수렴(OS 클립보드 단일 출처 규약 계승).
- [rows.rs](../../crates/nexa-gui/src/widgets/rows.rs): `RowSource::is_ghosted`
  (기본 false) 신설 — 페인트에서 해당 행 이름/셀 텍스트를 `text_dim`으로
  (리스트·타일 공통). [source.rs](../../crates/nexa-app/src/source.rs)
  `TreeSource`가 경로 대조로 구현(집합 비면 경로 조회 생략 — 핫패스,
  비-Windows는 cfg 격리 false).
- 테스트: rows `ghosted_row_paints_name_dim`(흐림 행 = text_dim·일반 행 =
  text 색 검증).

### ② Ctrl+V — 선택된 폴더 안으로 붙여넣기

- **요구**: 선택 없음/파일 선택 = 현재 폴더, **폴더 1개 선택 = 그 폴더 안**.
- [win.rs](../../crates/nexa-app/src/win.rs) `paste_dest()` 신설 — 활성
  패널 선택이 **정확히 1개이고 폴더**면 그 경로, 그 외(없음/파일/다중)는
  `root_path()`. Ctrl+V 핸들러가 기존 `root_path()` 고정 대신 이를 사용
  (컨텍스트 메뉴 pasteInto와 동일 판정 기준).

### ③ 파일 DnD 중 위/아래 자동 스크롤

- **요구**: 드래그로 목록 위/아래 가장자리에 닿으면 스크롤.
- [rows.rs](../../crates/nexa-gui/src/widgets/rows.rs)
  `drag_scroll_edge(y)` — 본문 상/하단 **한 행 높이** 존이면 ±1행
  (클램프·타일 그리드 행 공용). 호스트 폴링이 반복 호출 = 연속 스크롤.
- 테스트: `drag_scroll_edge_moves_one_row_within_edge_zone`.

### ④ DnD 호버 3초 — 탭 전환·접힌 폴더 펼침 (+설정)

- **요구**: 드래그 중 탭 위 3초 = 그 탭 활성화 · 폴더 이름 위 3초 = 펼침
  (펼친 폴더 안의 폴더에 다시 3초 = 중첩 펼침) · 시간은 설정(기본 3000ms).
- [dnd.rs](../../crates/nexa-app/src/dnd.rs): `DropHooks`에 `track`(DragEnter/
  DragOver 화면 좌표)·`leave`(DragLeave/Drop) 훅 추가 — State 접근은 기존
  규약대로 win.rs에 격리.
- [win.rs](../../crates/nexa-app/src/win.rs): `TIMER_DND`(100ms) 폴링 —
  OLE DragOver는 정지 커서에서 호출이 멎을 수 있어 `GetCursorPos`로 보강
  (드래그 중에만 무장·leave에서 해제). `dnd_track_update` = ① 커서 패널
  `drag_scroll_edge` ② 호버 후보(비활성 탭 `tab_index_at` > 접힌 폴더
  `is_collapsed_dir`) 판정 — `DndHover(Tab|Folder)` + 시작 시각 유지, 같은
  대상 `dnd_hover_ms` 경과 시 발동(탭 = `switch_tab`·폴더 = `hover_expand`,
  폴더는 행-경로 재검증으로 스크롤/재정렬 밀림 방어). 발동 후 리셋 —
  중첩 폴더는 새 대상으로 다시 3초 대기(요구 시나리오 그대로).
  탭 전환 시 `update_title/status` 길목 합류(watcher·세션 저장 동기).
- `hover_expand`는 **펼침 전용**(접기 없음)·Tree 모드 한정(Flat/Tiles 제외).
  테스트: `hover_expand_only_collapsed_dirs_in_tree_mode`.
- **설정 `dnd_hover_ms`**(기본 3000·200~10000 클램프):
  [config.rs](../../crates/nexa-app/src/config.rs) 직렬화/파싱/왕복 테스트 ·
  [prefs.rs](../../crates/nexa-app/src/prefs.rs) "파일 전송" 카테고리
  `F_DND_HOVER` Number 행 · lang 3종(en/ko/ja) `pref.dndHover(.desc)` ·
  적용은 다음 드래그부터(재시작 불요).
