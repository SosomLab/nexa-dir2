## Chocolatey 포터블 패키지 추가 — `nexa-dir.portable` (사용자 요청)

> **사용자**: "윈도우 pc에서 choco 패키지 배포를 하려고 해 / portable을 0.8.1 버전으로".

- **기존 0.8.1 릴리스를 그대로 재사용한다** — 새 태그·재빌드 불필요. Chocolatey가
  막는 건 *같은 ID+같은 버전* 재push뿐이고, `nexa-dir.portable`은 신규 ID라
  그 첫 버전이 0.8.1이 된다. 참조 자산(`NexaDir-0.8.1-win-x64.exe`)도 이미 존재.
- **ID = `.portable` 접미사**(Chocolatey 관례 `git.install`/`git.portable`).
  엄밀히는 설치형도 `nexa-dir.install`이어야 하나 **모더레이션 중 개명은 위험**해
  `nexa-dir` 유지 — 문서에 근거 기록.
- 디렉터리 분리: `packaging/chocolatey/<id>/{nuspec,tools}` (기존 파일 git mv).
- 포터블 설치 = `Get-ChocolateyWebFile`로 `tools\NexaDir.exe` 배치 → choco 자동 shim.
  **`NexaDir.exe.gui` 빈 파일** 동봉 = GUI 앱이라 shim이 종료를 기다리지 않게 하는 마커.
- **한계 — 제거 시 데이터 소멸**: `data\`가 패키지 폴더 안이라 `choco uninstall`이
  폴더째 삭제. winget portable과 동일 성질(§8) — 설명·문서에 명시하고 보존이 필요하면
  설치형을 쓰도록 안내.
- release.yml = 2패키지 루프로 확장(자산별 체크섬 주입 → pack → push).
- **`pack-and-push.ps1` 신설**(Windows 전용) — 태그 소진 버전을 뒤늦게 올리는 경로.
  자리표시자 치환은 `try/finally`로 **원본 복원 보장**(치환된 채 커밋되는 사고 방지).
- `.gitignore`에 `installer/out/`·`packaging/chocolatey/out/` 추가(수동 실행 산출물).

## Chocolatey 배포 채널 등록 (`feat/chocolatey-packaging`)

> **사용자**: "github release까지 진행된 상태 / chocolatey 배포할 수 있도록 등록 진행해줘".

- **`packaging/chocolatey/`** 신설 — `nexa-dir.nuspec` + `tools/`(install·uninstall·VERIFICATION).
  설치형 채널의 래퍼일 뿐 **새 산출물은 만들지 않는다**(설계 = [21 §7](../21-distribution.md)).
- **바이너리 미동봉 방식** 채택: nupkg는 스크립트만, 설치 시 Release의
  `NexaDir-Setup-<버전>.exe`를 SHA-256 검증 후 다운로드. 근거 = PolyForm NC(비-FOSS)는
  커뮤니티 저장소 재배포 규정상 공식 URL 다운로드가 정석 + nupkg 수 KB 유지.
- **함정 1 — 사용자별 설치로 새는 문제**: choco는 관리자로 돌지만 `nexa.iss`의
  `PrivilegesRequired=lowest`는 그것만으로 *사용자별* 설치로 판정 → 관리자 계정의
  `%LOCALAPPDATA%\Programs`에 설치된다. 해결 = 무인 인자에 **`/ALLUSERS`** 명시 +
  이를 받으려면 `PrivilegesRequiredOverridesAllowed`에 **`commandline` 추가**(iss 수정).
- **함정 2 — `secrets` 컨텍스트**: step 레벨 `if`에서 `secrets.X`는 인식되지 않는다
  (IDE 진단이 잡아줌). job 레벨 `env`로 승격 후 `env.CHOCO_API_KEY != ''`로 게이트.
- release.yml 확장: 설치형 빌드 직후 체크섬 주입 → `choco pack` → 시크릿 있을 때만
  `choco push`(없으면 팩까지만 + 아티팩트 첨부 = 수동 push 여지).
- **잔여(사용자 수행)**: community.chocolatey.org 계정·API 키 발급 → 저장소 시크릿
  `CHOCO_API_KEY` 등록 → 다음 태그 push. **첫 패키지는 모더레이션 심사**(수일~2주) 대기.
