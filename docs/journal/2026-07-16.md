## X-17 가상 최상위 "내 PC" (`feat/x17-my-pc`)

> **사용자 요청**: C:\ 위로 이동 불가 → 가상 최상위 영역 + 드라이브 간 이동
> (탐색기 "내 PC" 스크린샷). 원본 확인: 드라이브 뷰 미구현(FR-A5 사이드바 계획만·
> 경로 바 `C:`→`C:\` 규약뿐) — 신규 설계.

- **센티널 설계**: `nexa_vfs::MY_PC = "::PC::"`(콜론 = 실 경로 불가 문자 — 충돌 0).
  드라이브 항목 **이름 = `C:\`(절대 경로)** → nexa-tree의 `dir.join(name)`이 부모를
  대체해 노드 경로가 곧 실 드라이브 루트 — **진입·선택·셸 아이콘·펼침이 기존
  파이프라인 그대로 동작**(특수 케이스 최소화가 핵심).
- **vfs**: `drive_entries()` = `A:\`~`Z:\` metadata 프로브(std만 — Win32 API 불요,
  크레이트 플랫폼 중립 유지). **tree**: 열거부만 분기(가상 루트 = 드라이브 목록).
- **panel**: `nav_up` — 드라이브 루트(parent 없음)에서 **내 PC 진입 + 떠난 드라이브
  자동 선택**(G-7 일관), 내 PC = 최상위(무동작), 뒤로/앞으로 왕복. 세션에 `::PC::`
  저장/복원도 동일 경로(Tree::open 경유)로 자연 처리.
- **표시**: 탭 제목·경로 바(단일=현재 세그먼트라 클릭 무동작)·창 타이틀 =
  `nav.mypc`(내 PC/This PC) · 종류 셀 = `kind.drive` · **수정일 0 = 빈 셀**(1970 방지).
- watcher는 가상 루트 감시 실패 = None(기존 F5 폴백 규약 그대로).
- 테스트 3건 추가(173 green): vfs 열거·tree 가상 루트(경로=실 루트·index_of_path)·
  panel 위로→선택→뒤로 왕복. **β**: 볼륨명("로컬 디스크 (C:)")·용량/사용량 바
  컬럼 데코(GetVolumeInformationW·GetDiskFreeSpaceExW — Win32 데코 레이어).
