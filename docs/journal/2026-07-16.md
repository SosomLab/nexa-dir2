## ctl 2호 — 글꼴 입력 fontbox (`feat/ctl-fontbox`)

> **사용자 요청**(WT 글꼴 피커 스크린샷): 창 클릭 = 설치 글꼴 목록(스크롤·키보드/
> 마우스 탐색·선택), 선택 시 `,` 구분자 있으면 뒤에 추가·없으면 뒤에 붙이기,
> 글꼴 검색 타입어헤드 = **마우스 커서 위치에 입력값 보기** + 매칭 위치 이동.

- **`ctl::fontbox`**(NexaFontBox): 자식 EDIT + **드롭다운 팝업**(NexaFontDrop —
  WS_EX_NOACTIVATE라 에디트 포커스 유지·오너드로 LISTBOX) + **커서 HUD**(NexaFontHud).
  - 설치 글꼴 = EnumFontFamiliesExW 1회(OnceLock — '@' 세로쓰기 제외·ci 정렬).
  - 목록 항목을 **그 글꼴로 미리보기 렌더**(항목별 HFONT 지연 생성·팝업 파괴 시 해제).
  - 키보드: ↓(닫힘) = 열기, ↑/↓/PgUp/PgDn = 이동, Enter = 확정, Esc = 닫기 —
    내부 EDIT를 user32 원시 서브클래스(GWLP_WNDPROC — comctl32 불요)로 라우팅.
    마우스: hover = 선택 이동(LB_ITEMFROMPOINT)·클릭 = 확정·MA_NOACTIVATE.
  - **선택 규칙**(단위 테스트): 마지막 `,` 뒤 조각이 선택 글꼴의 접두사(검색 입력)면
    교체, 완결 이름이면 `, `로 뒤에 추가, 빈 입력이면 그대로 — 폴백 체인 자연 구축.
  - **타입어헤드 HUD**: 입력 중 현재 조각을 "찾기: X"로 커서 옆에 표시(accent 테두리 —

  - 계약: searchbox와 동일(텍스트 위임 + EN_CHANGE) + **확정 = EN_KILLFOCUS 재발행**
    → prefs 기존 즉시 적용(0x0200) 경로 그대로.
