- **① 셀 단위 렌더**: 런 단위 단일 레이아웃이 원인 — 폴백 글꼴(한글=Malgun 등·Nerd

  fill, **문자는 셀 x에 개별 배치**(전각=2셀). 단일 글리프 레이아웃 캐시(`mono_glyphs`)
  로 프레임당 생성 비용 제거.
- **② 드래그 선택**: TermState에 sel(절대 라인·열)/grid 캐시 — 프레스=앵커·드래그 확장
  (반전 하이라이트)·그리드 밖 상/하=1줄 자동 스크롤+60ms 반복(TIMER_TERM_SEL)·업 후 유지.
- **③ 스크롤백 보기**: `view_off`(0=라이브) — 터미널 위 휠 3줄/노치·보기 중 새 출력은
  보던 위치 고정·입력 시 하단 스냅·캐럿은 가시 범위만(전부 WT 규약).
- **④ 클립보드**: Ctrl+C=선택 있으면 `get_text` 복사(CRLF)+해제·없으면 ^C 인터럽트,

- **⑤ 글꼴**: settings `term_font`(기본 Consolas) — DWrite가 사용자 설치 글꼴(D2Coding
  등)을 시스템 컬렉션에서 해석·미설치는 Consolas 폴백. **가능** — 지정 글꼴에 없는

  원본식 명시 폴백 체인(1순위→2순위)은 IDWriteFontFallbackBuilder 필요 — X-3 후속.
- α: 스크롤백 800 상한 도달 시 절대 인덱스 이동(선택/보기 미세 드리프트)·더블클릭

## 실기 QA 배치 2 (`fix/m4-qa-batch2`) — 프리즈 근본 원인 해소 외 5건

> **사용자 QA**: ① Downloads/Documents 진입·탭 추가 후 "응답 없음"(수십 초 후 회복)
> ② 경로바 편집 Ctrl+C/V/X 무반응 ③ 선택 상태 더블클릭이 리네임으로 오진입
> ④ 휠이 마우스 위치와 무관하게 활성 패널만 스크롤 ⑤ 탐색기처럼 .lnk 확장자 숨김 요청.

- **① 프리즈 원인**: 폴더 크기가 아니라 **파일별 아이콘 로드의 UI 스레드 동기 호출**.
  exe·lnk 등은 실제 파일을 여는 SHGetFileInfoW인데, 다운로드 파일(Mark-of-the-Web)은
  첫 접근에 Defender 실시간 검사, OneDrive 자리표시자는 하이드레이션이 걸려 수십 초
  블로킹. **워커 스레드**(지연 생성·COM STA)로 이전, 결과는 WM_APP_ICON(Box 소유권
  이전)으로 회수. 타입 아이콘(레지스트리 조회·파일 접근 0)은 동기 유지. in-flight
  큐 표시로 중복 로드 차단·같은 키 대체 시 이전 HICON 해제. 원본 ShellIconCache는
  WinRT GetThumbnailAsync(비동기)라 이 문제가 없었음 — 이식 시 동기화가 결함.
- **② 편집 클립보드**: clipboard.rs에 CF_UNICODETEXT `write_text`/`read_text` 신설,
  EditState에 `selected_text`/`cut_selection`/`insert_str`(선택 대체) — 경로바 편집·
  인라인 리네임 양쪽에 Ctrl+C/X/V 배선(붙여넣기=첫 줄·제어 문자 제거).
- **③ 리네임 지연**: 느린 재클릭 확정을 즉시가 아닌 **GetDoubleClickTime() 지연**
  (TIMER_RENAME) — WM_LBUTTONDBLCLK가 취소(더블클릭=열기 우선)·새 클릭도 무효화.
- **④ 휠 라우팅**: WM_MOUSEWHEEL/HWHEEL lparam(화면 좌표)→클라이언트 변환 →
  `wheel_target` = 마우스 아래 패널(밖이면 활성 폴백) — hover 스크롤.
- **⑤ .lnk 숨김**(NeverShowExt): 이름 컬럼 표시만 숨김(UTF-8 경계 가드)·확장자 컬럼
  유지·리네임 확정 시 .lnk 자동 복원(win.rs apply_rename).
- 검증: 151 green(신규 3 — in-flight·insert 대체·display_name)·clippy 0·릴리스 빌드.
