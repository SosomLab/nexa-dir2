> **사용자 QA**: 아이콘 비동기화 후에도 Documents/Downloads 이동·경로바 브레드크럼
> 이동에서 5-10초 멈춤 잔존. + 터미널 캐럿 깜빡임 요청.

  UI 블로킹 실측 → 홈→Documents에서 **12,009ms** 무응답 재현.
- **원인**: `DirWatcher::drop`의 `CloseHandle`. **비 OVERLAPPED 핸들은 파일 객체 잠금이
  직렬화**되어, 대기 중인 동기 ReadDirectoryChangesW가 반환할 때까지(=**이전 폴더에
  변경이 생길 때까지**) CloseHandle이 블로킹. 이동=이전 watcher drop → UI 스레드 정지.
  조용한 폴더(OneDrive Documents)일수록 오래 멈춤·이동 후엔 정상·복귀 시 재발 —
  전부 부합. 초판 주석의 "핸들 닫기 → 대기 호출이 풀림" 가정이 오류였음.
- **수정**: OVERLAPPED RDCW + 중지 이벤트 대기(WaitForMultipleObjects) — drop은
  **SetEvent만**(논블로킹), IO 취소(CancelIoEx)·핸들 정리는 스레드 몫. 스레드에는
  중지 이벤트 **복제 핸들**(DuplicateHandle — 핸들 값 재활용 경쟁 차단).
- **검증(동일 자동화)**: 홈→Documents 12,009ms→**6ms** · Documents→홈(조용한 OneDrive
