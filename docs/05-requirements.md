## 2. 비기능 요구 (NFR) — 예산 게이트 ★

> **게이트 규율**: 각 마일스톤 종료 시 실측. **B1~B3 초과 상태로는 main 병합 금지**(측정 방법 → [18](18-build-and-test.md) §5).

| # | 항목 | 예산 | 비고 |
| --- | --- | --- | --- |
| **B1** | 유휴 RSS(Working Set) | **≤ 30MB** (목표 20MB급) | 창 1개·탭 2개·10k 폴더 로드 후 유휴 5분 |
| **B2** | 단일 exe 크기 | **≤ 10MB** (release, strip) | 리소스(아이콘·기본 lang) 임베드 포함 |
| **B3** | 임포트 DLL | **OS 인박스만** | dumpbin/imports 검사 — 재배포 런타임·서드파티 dll 0 |
| B4 | 콜드 스타트 | < 200ms (창 표시까지) | |
| B5 | 대용량 | 100k 항목 첫 렌더 < 150ms·스크롤 60fps | 원본 G1·10만 노드 벤치 계승 |
| B6 | 상주 규율 | 백그라운드 CPU ~0%·유휴 트림·오류 격리 | 원본 NFR 계승 |

## 3. 제약 (Constraints)

- **C1 언어/스택**: unmanaged **Rust 단일 언어**. GC 런타임·프레임워크(WinUI/Electron/WebView) 금지. → [DR-1](10-decision-record.md)
- **C2 의존성 라이선스**: 퍼미시브(MIT/Apache/BSD/ISC/MPL-2.0) 온리, GPL/AGPL 금지 — 원본 규율 계승. Slint(GPL/유료 이중) 배제 근거.
- **C3 의존성 수**: 외부 crate는 **기본 0 지향**, 추가는 건별로 [10](10-decision-record.md)에 기록(예산 B2 영향 평가 동반). `windows`/`windows-sys` crate는 승인.
- **C4 코어 재사용**: 원본 `nexa-core`/`nexa-vfs`/`nexa-tree`를 rlib 직접 링크(cdylib/FFI/ABI 폐지). 개선은 가능하되 의미 변경은 ADR.
- **C5 Windows API 정책**: Win32/COM 직접 호출(`windows` crate). Windows 10 1809+ 지원(ConPTY 하한, 원본과 동일).
- **C6 라이선스**: 본체 PolyForm Noncommercial 1.0.0(원본 DR-5 계승).
