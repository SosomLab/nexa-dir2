# 설계 결정

Nexa Dir의 정체성을 규정하는 예산 게이트, 확정 결정(DR), 핵심 ADR을 정리합니다. 2026-07-11 확정, 이후 변경은 새 ADR/journal로 추적합니다.

---

## 예산 게이트 — 프로젝트의 정체성

목표(Goal)와 예산(Budget)이 분리되지 않습니다. **예산 자체가 정체성**이며, 마일스톤 종료마다 실측해 초과 상태로는 main에 병합하지 못합니다.

| 게이트 | 기준 | 측정 방법 |
| --- | --- | --- |
| **B1 유휴 RSS** | ≤ 30MB (목표 20MB급) | 앱 기동 → 10k 폴더 로드 → 유휴 5분 → `WorkingSet64` 3회 중앙값 |
| **B2 exe 크기** | ≤ 10MB | `target/release/nexa-app.exe` 크기, CI에서 초과 시 fail |
| **B3 임포트 DLL** | **OS 인박스만** | `dumpbin /imports`, 인박스 외 발견 시 fail. 화이트리스트 스크립트 `scripts/budget-b3.ps1`이 CI·로컬 공용 SSOT |
| 외부 crate | 기본 0 지향 | 추가는 건별 원장 기록 (DR-8) |

유휴 RSS가 트림 후 사실상 0MB대로 떨어지는 이유는 상주 규율(유휴 60초·최소화 시 작업집합 반납, 다음 페인트 지연 재적재) 덕분입니다.

## 확정 결정 (DR-1 ~ DR-8)

| # | 한 줄 설명 |
| --- | --- |
| **DR-1** | **올 러스트 단일 바이너리.** Win32(windows-rs) 직접 호출 + 커스텀 드로잉. 관리 런타임·UI 프레임워크·WebView **금지**. → ADR-0001 |
| **DR-2** | **예산 게이트.** B1 유휴 RSS ≤30MB · B2 exe ≤10MB · B3 임포트 = OS 인박스만. 마일스톤 병합 게이트 |
| **DR-3** | **배포 2채널** (07-16 개정). 포터블 단일 exe **기본** + 설치형 exe(Inno Setup) **보조**. 영속물 = exe 옆 `data\`, 쓰기 불가 위치면 `%LOCALAPPDATA%\NexaDir\data` 폴백. **무서명 유지 확정**(SmartScreen 경고 감수 — 인증서 확보 시 서명 추가) |
| **DR-4** | **코어 rlib 이식.** 원본 nexa-core/vfs/tree를 rlib 직접 링크로(cdylib/C ABI 폐지) |
| **DR-5** | **원본 M1 기능 패리티** + 디자인 규약 계승(고밀도·다크 기본·키보드 우선·테마 토큰) |
| **DR-6** | **라이선스.** 본체 PolyForm Noncommercial 1.0.0 + 의존성 퍼미시브 온리(GPL/AGPL 금지 — Slint 배제 근거) |
| **DR-7** | **플러그인/미리보기** (07-14 개정). .NET SDK 비이관 유지 + **Starlark 임베드 플러그인 도입**(내장 미리보기는 폴백 존치). WASM 보류 |
| **DR-8** | **외부 crate 기본 0 지향.** 추가는 원장에 건별 기록 + 예산 영향 평가 |

## 핵심 ADR (Architecture Decision Records)

### ADR-0001 — 스택 선택
후보 6종(올 러스트 / Slint / egui / C++ 혼합 / Delphi / 원본 유지)을 단일 exe·유휴 RSS·코어 재사용·라이선스로 비교해 **"올 러스트 + Win32 + 커스텀 드로잉"**을 채택. Slint는 GPL/로열티(퍼미시브 위반)로, egui는 GPU 상시 리페인트(B1 위반)로 기각. 트레이드오프는 **UI 전면 자체 구현**(탭·스크롤바·메뉴·인라인 편집기…)이라는 최대 공수를 감수하는 것.

### ADR-0002 — 렌더링
Windows 11 실기·100k 행 벤치에서 GDI `ExtTextOutW` vs **DirectWrite GDI interop**(`IDWriteBitmapRenderTarget` + `IDWriteTextLayout`)을 같은 창에서 전환 비교 → DirectWrite가 스크롤 벤치 **−28%**, RSS +4.1MB(예산 내), 한글 폰트 폴백 일관성 + TextLayout(트리밍·히트테스트)이 컬럼·타입어헤드 요구와 직결되어 **DirectWrite 채택**. GPU 스왑체인은 계속 금지(B1 위반 경로).

### ADR-0003 — 셸 컨텍스트 메뉴
우클릭 = 클래식 `IContextMenu`(COM) + HMENU 호스팅(7-Zip·Git·보내기·속성 등 셸 확장 표시). **dir2 특이점**: 원본이 comctl32 서브클래스로 하던 메뉴 메시지 포워딩을 **자기 wndproc이 직접** 처리 → **comctl32 임포트 회피(B3 무증가)**. COM 인터페이스는 windows-rs 제공(수동 vtable 0).

### ADR-0004 — 미리보기 플러그인
"EXIF 미리보기를 Python처럼 개발"하고 싶다는 요구에 **Starlark 임베드**(Python 유사 문법·샌드박스·정적 링크 = 단일 exe·Apache-2.0)를 채택. 계약 = `data\plugins\*.star`(확장자 선언 + preview 함수), 다중 매치 시 도크 우상단 공급자 콤보. `starlark` crate는 exe 크기 실측 후 승인 예정. 현재는 **계획 단계**이며 내장 미리보기(텍스트/이미지)만 동작합니다 — **[플러그인 개발](개발-플러그인)** 참고.

## 개발 방법론

1. **수직 슬라이스** — 관찰·테스트 가능한 얇은 끝단(코어 → gui → 창) 단위로 자른다.
2. **작게·순차로 · 초안 먼저(확장은 별도 커밋) · main 항상 green · 단위 = 커밋 1개 · 테스트 동반.**
3. 큰 미지수는 **스파이크**(버릴 수 있는 실험)로 먼저 검증.
4. **원본 참조 우선** — 기능 설계 전 원본 문서·코드 확인(재발명 금지), 이식 커밋에 원본 경로 명기.
5. **Conventional Commits** · 큰 단위 = 브랜치 / 세부 = 커밋 · push는 명시 요청 시에만.

> 빌드·테스트·CI 절차는 **[빌드와 테스트](개발-빌드와-테스트)** 문서를 참고하세요.
