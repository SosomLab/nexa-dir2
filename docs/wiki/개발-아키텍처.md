# 아키텍처

Nexa Dir 2는 **올 러스트 단일 바이너리**입니다. 원본(Rust cdylib + WinUI 3/C#) 대비 핵심 변화는 **언어 경계·FFI·관리 런타임 제거**입니다.

---

## 크레이트 층 구조

```
┌─────────────────────────────────────────────────┐
│ nexa-app (bin)  창·메시지 루프·레이아웃·명령 배선     │
├─────────────────────────────────────────────────┤
│ nexa-gui        위젯(리스트·탭·경로바·메뉴)·렌더러     │  ← M1 분리
│                 (Win32 창 1개 + 전면 커스텀 드로잉)   │
├──────────────┬──────────────┬───────────────────┤
│ nexa-tree ★  │ nexa-ops     │ nexa-shell        │
│ 트리·선택·정렬 │ 전송·Undo    │ COM(셸메뉴·DnD·     │
│ (원본 이식)   │ (M3 신규)    │ 휴지통·아이콘)      │
├──────────────┴──────┬───────┴───────────────────┤
│ nexa-vfs (원본 이식) │ nexa-term (ConPTY·VT, M4) │
├─────────────────────┴───────────────────────────┤
│ nexa-core (공용 타입, 원본 이식)                    │
└─────────────────────────────────────────────────┘
        전부 rlib 정적 링크 → 단일 exe (FFI/ABI 없음)
```

| 크레이트 | 출처 | 내용 |
| --- | --- | --- |
| `nexa-core` | 원본 이식 | 공용 타입(FileKind 등), 의존성 0 |
| `nexa-vfs` | 원본 이식 | 로컬 스트리밍 열거·Provider trait |
| `nexa-tree` ★ | 원본 이식 | 가시 노드 평면 스트림·교차 선택·정렬·타입어헤드 |
| `nexa-app` | 신규 | bin. Win32 창·메시지 루프·조립(비-Windows는 스텁) |
| `nexa-gui` | 신규 | 위젯 트리·커스텀 드로잉 렌더러·입력 라우팅·테마 토큰 |
| `nexa-ops` | 신규 | 전송 엔진·Undo/Redo·휴지통 |
| `nexa-term` | 신규 | ConPTY 세션·VT 파서/스크린 버퍼 |
| `ctl/`(nexa-app 내) | 신규 | Win32 커스텀 컨트롤 라이브러리(Nx 컨트롤 14종) → [컴포넌트 개발](개발-컴포넌트) |

**핫패스 전부 러스트 단일 프로세스** — 원본의 P/Invoke 마샬링·행당 왕복 같은 경계 비용이 소멸했습니다. 순수 로직 성질을 유지해 **맥에서도 테스트**할 수 있습니다.

## 렌더링 파이프라인 — 창 1개 + 전면 커스텀 드로잉

Double Commander/Everything 체급의 메모리는 GPU 컴포지팅 프레임워크 없이 Win32 창에 직접 그릴 때 달성됩니다.

- **창/입력**: `RegisterClassW` + `CreateWindowExW` 창 **1개**. 위젯(탭·메뉴·스크롤바)은 전부 논리 객체(자식 HWND 최소화). PerMonitorV2 DPI.
- **페인트**: `WM_PAINT` **더블 버퍼**(메모리 DC). **가시 영역만** 그립니다 — `nexa-tree`의 가시 행 평면 스트림이 이 모델에 정확히 부합(행 = `rect × draw(dc)`). 프레임 비용이 창 높이에만 비례.
- **텍스트**: **DirectWrite GDI interop**(ClearType + 시스템 폰트 폴백). 가시 행 레이아웃 캐시로 최적화.
- **AA(곡선 UI)**: **GDI+ 백엔드**(`gdipctx` — gdiplus.dll 인박스)를 `DrawCtx`의 유일한 GDI+ 접점으로 두어 외부 crate 0 유지. 아이콘은 **SVG 서브셋 파서**로 요청 크기 즉석 래스터.
- **금지**: D3D 스왑체인 상시 보유·상시 리페인트 루프 = B1 예산 위반 경로.

## 스레딩 모델

- **UI 스레드 1개**(메시지 루프) + **워커**(디렉터리 열거·전송·watcher·셸 아이콘). 워커 → UI 통지는 `PostMessage`로 단일화(세대 가드).
- 취소는 세대 카운터/`AtomicBool`. 패널·탭 상태의 진실원천은 코어(`nexa-tree` 핸들).
- UIA 접근성 콜백은 임의 스레드에서 오므로 불변 스냅샷(Arc 공유) 모델로 안전 처리.

---

## 영속·리소스 (포터블 규율)

- 영속물(설정·세션·언어팩) = **exe 옆 `data\`** 단일 원천. serde 대신 key=value 텍스트 + 원자적 쓰기(tmp → rename).
- 아이콘·기본 언어팩(en/ko)은 `include_str!`/`include_bytes!`로 exe에 임베드 → 외부 파일 0. 사용자 언어팩은 `data\lang\` 키 단위 오버라이드.
