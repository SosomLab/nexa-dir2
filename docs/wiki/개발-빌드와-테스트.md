# 빌드와 테스트

Nexa Dir 2는 **맥 일상 개발 / Windows 실행·신뢰 원천** 모델로 개발됩니다.

---

## 개발 환경

| 부문 | macOS | Windows |
| --- | --- | --- |
| 코어 크레이트(순수부) | ✅ build·test | ✅ |
| Windows 전용 코드(gui/shell/term) | ✅ `cargo check --target x86_64-pc-windows-msvc`(타입 검증) | ✅ build·실행 |
| 실행·실기 QA·예산 실측 | ✗ | ✅ PC/VM/CI |

원본 대비 결정적 개선: **WinUI XAML 컴파일러 의존이 사라져 UI 코드까지 맥에서 타입 검증**됩니다. WinUI 시절의 "맥은 앱 빌드 불가" 규율은 이제 **실행 검증**에만 남습니다.

---

## 로컬 빌드

```sh
# 코어 테스트(어느 OS나)
cargo test --workspace

# UI 코드까지 타입 검증(맥에서)
cargo check --target x86_64-pc-windows-msvc

# 릴리스 빌드(Windows)
cargo build --release -p nexa-app
# → target/release/nexa-app.exe
```

빌드 프로파일은 정적 CRT · LTO fat · panic abort · 심볼 스트립으로 exe를 최소화합니다.

## 예산 게이트 (로컬 실행)

push 전, 특히 새 임포트 DLL이 생기는 변경에는 로컬에서 게이트를 돌립니다.

```sh
# B3 임포트 화이트리스트 검사(CI와 동일 스크립트)
pwsh scripts/budget-b3.ps1
```

- **B2**(exe ≤ 10MB) · **B3**(임포트 = OS 인박스만)는 CI와 **동일한 단일 출처**(`scripts/budget-b3.ps1`)로 검사합니다.
- 인박스 DLL이 늘어나는 변경은 근거와 함께 화이트리스트에 등재해야 합니다(커밋 메시지에 기록).
- 예산 기준의 배경은 → [설계 결정](설계-결정)의 예산 게이트 절.

---

## CI (GitHub Actions)

| 워크플로 | 실행 |
| --- | --- |
| `core` (ubuntu + macos) | `cargo test` |
| `windows` (windows-latest) | `cargo build --release` + `cargo test` + **예산 게이트**(B2 · B3) |
| `release` (태그 push) | test → 예산 게이트 → 포터블 exe + Inno Setup 설치형 2종을 GitHub Release 자동 첨부 |

---

## 릴리스 절차

```sh
git tag 0.7.0 && git push origin 0.7.0
# → Actions "Release" 실행 → 게이트 통과 시 releases 에 exe 2종 첨부
```

- 태그 규약: `X.Y.Z`(또는 `vX.Y.Z`).
- 산출물: `NexaDir2-<버전>-win-x64.exe`(포터블) + `NexaDir2-Setup-<버전>.exe`(Inno Setup 설치형).
- 코드 서명은 현재 **무서명**(SmartScreen 경고 감수) — 인증서 확보 시 서명 단계를 추가할 예정입니다.
