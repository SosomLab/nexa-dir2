# 프로젝트 소개

**Nexa Dir 2**는 한 줄로 이렇게 정의됩니다.

> 원본 Nexa Dir의 기능 경험을 **포터블 단일 exe · 초저메모리 · 순수 네이티브(올 러스트)**로 다시 짓는 Windows 파일 탐색기.

## 왜 만들었나 — 문제는 기능이 아니라 "스택"이었다

원본 [Nexa Dir](https://github.com/SosomLab/nexa-dir)은 **Rust 코어 + WinUI 3/C#** 스택으로 만든, 기능적으로 성숙한 탐색기입니다. 그러나 포터블·경량 에디션을 향할 때 스택 구조 자체가 만든 세 가지 벽이 실측으로 드러났습니다.

| 문제 | 원본 실측 | 원인 |
| --- | --- | --- |
| **메모리** | 유휴 RSS **60MB+** | .NET CLR + WinAppSDK/WinUI 컴포지션이 메모리 하한을 결정 |
| **배포** | self-contained ≈ **64MB**, dll 수십 개 | WinUI 3의 XAML/PRI 리소스가 단일 exe 게시에서 반복 결함 |
| **의존** | WinAppSDK 런타임 + .NET 재배포 동반 | "포터블 단일 exe" 목표와 정면 충돌 |

핵심 진단은 **한계의 원인이 기능이 아니라 관리(managed) 런타임 + UI 프레임워크 계층**이라는 것이었습니다. 그래서 이 프로젝트는 동일한 기능을 **unmanaged 올 러스트 스택**으로 재구축합니다. 원본은 폐기 대상이 아니라 **설계·기능 명세·실측 교훈의 원천(SSOT)**으로 계속 참조합니다. 두 에디션은 병행합니다 — 원본 = 설치형·WinUI, 본 프로젝트 = 포터블·네이티브.

---

## 4대 설계 원칙

1. **예산이 정체성 (Budget-first)** — 유휴 RSS ≤30MB · 단일 exe ≤10MB · 외부 DLL 0(OS 제공 제외). **마일스톤마다 실측 게이트**로 강제합니다. 예산 초과 상태로는 main에 병합하지 않습니다.
2. **진짜 포터블** — 설치·런타임·레지스트리 없이 exe 1개. 영속물은 exe 옆 `data\` 폴더에.
3. **성능 최우선 (Native-first)** — 콜드 스타트 <200ms · 100k 항목 첫 렌더 <150ms · 스크롤 60fps.
4. **원본 패리티** — 인라인 트리 + 폴더 교차 다중 선택(플래그십)·듀얼 패널/탭·파일 조작·터미널을 순차 재현.

자세한 예산 규칙과 결정 배경은 **[설계 결정](설계-결정)** 문서를 참고하세요.

## 경쟁 지형 속 차별점

| 제품 | 스택 | 유휴 RSS | 배포 |
| --- | --- | --- | --- |
| Double Commander | Free Pascal/LCL | <20MB | 포터블 zip (UI 노후) |
| Total Commander | Delphi/Win32 | ~15MB | 유료·구식 |
| Everything | C++/Win32 | <20MB | 단일 exe (검색 전용) |
| One Commander | C#/.NET WPF | 100MB+ | 관리 런타임 체급 |
| 원본 Nexa Dir | Rust + WinUI 3 | 60MB+ | setup.exe ≈64MB |
| **Nexa Dir 2** | **올 러스트/Win32** | **목표 ≤30MB** | **단일 exe** |

> **차별화 공식**: *원본의 플래그십 UX(인라인 트리 + 폴더 교차 선택)* × *Win32 커스텀 드로잉 체급의 예산* × *단일 exe*. 이 조합을 제공하는 제품은 없습니다.

## 무엇을 할 수 있나

Nexa Dir 2는 이미 일상적인 파일 탐색·관리에 필요한 기능 대부분을 갖추고 있습니다.

- **보기 3종** — 트리(인라인 펼침)·플랫·타일, 5컬럼·다중열 정렬·컬럼 드래그 재배열
- **듀얼 파일 패널 + 패널별 탭** — 잠금/고정/복제·세션 복원
- **폴더 교차 다중 선택** — 인라인 펼친 상태에서 서로 다른 폴더 항목을 동시에 선택(플래그십)
- **파일 조작** — 복사/이동/삭제/이름변경/새로 만들기·진행률·충돌 처리·Undo/Redo·휴지통·드래그앤드롭
- **하단 도크** — 정보 / 미리보기(텍스트·이미지) / 내장 ConPTY 터미널
- **일괄 이름 변경** — 동작 파이프라인(치환·정규식·대소문자·삽입·연번·날짜)·실시간 미리보기·프리셋
- **셸 통합 컨텍스트 메뉴** · **퀵 런처 바** · **테마/언어(한·영) 즉시 전환** · **커스터마이즈 가능한 도구 모음/컬럼/컨텍스트 메뉴**

전체 기능은 **[기능 개요](기능-개요)**에서 탐색할 수 있습니다.
