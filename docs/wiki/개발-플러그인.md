# 플러그인 개발 (미리보기)

> ⚠️ **현재 상태 요약**: 미리보기 **플러그인(Starlark) 시스템은 계획 단계**입니다. 코드베이스에 Starlark 런타임·`PreviewProvider` 추상화·`.star` 로딩은 아직 없습니다. 현재 동작하는 것은 **내장(built-in) 미리보기**(텍스트·이미지)뿐입니다. 이 문서는 "현재 상태"와 "로드맵"을 명확히 구분합니다.

---

## 1. 왜 플러그인인가 (설계 결정 — ADR-0004 / DR-7)

"EXIF 정보 미리보기를 Python처럼 개발하고 싶다"는 요구에서 출발했습니다.

### 왜 Starlark인가

- **Starlark** = Python 문법의 결정적·샌드박스 부분집합(Bazel/Buck2 계열, Apache-2.0). 인터프리터를 **정적 링크로 임베드**.
- 채택 근거:
  - Python 유사 문법 → "Python처럼 개발" 요구 충족.
  - **샌드박스** — 파일/네트워크 기본 차단, 호스트가 명시적으로 준 API만 접근.
  - **결정적 실행 + 정적 링크** → 단일 exe 원칙(DR-3) 유지.
  - 퍼미시브 라이선스(DR-6).

### 기각·보류된 대안

- **WASM** — 샌드박스는 우수하나 "Python으로 개발" 요구와 거리가 있고 런타임 크기 부담 → **보류**(Starlark로 부족해지면 재검토).
- **.NET SDK 이관**(원본 방식) — 관리 런타임 필요 → DR-1(unmanaged)·단일 exe와 상충하여 기각.
- **CPython 임베드** — 런타임 수십 MB → B2 예산(exe ≤10MB) 초과·샌드박스 부재로 기각.

### 내장 미리보기와의 관계 (폴백 규약)

내장 텍스트/이미지 미리보기는 **폴백 안전망으로 존치**됩니다. 플러그인이 0개여도 기본 미리보기는 항상 동작합니다. 우선순위(계획): **설정 오버라이드 > 콤보 선택 > 플러그인 선언 매치 > 내장 폴백**.

## 2. 현재 구현 상태 (내장 미리보기)

현재 미리보기는 앱 내부의 함수 하나(`preview_content`)로 처리됩니다. 지원 범위:

| 대상 | 동작 |
| --- | --- |
| **텍스트** | 파일 첫 16KB · UTF-8 lossy · 최대 200줄 · 탭 4칸 |
| **이미지** | 9종(png/jpg/jpeg/bmp/gif/ico/tif/tiff/webp) · WIC 디코드 · 비율 유지 가운데 렌더 |
| **빈 파일** | `(빈 파일)` |
| **이진 파일** | 첫 1KB에 NUL 바이트 → `이진 파일 — 텍스트 미리보기 없음` |
| **읽기 실패** | `읽기 실패` |

도크 위젯(`InfoDock`)은 "라인 또는 이미지"만 그리는 단순 뷰이고, 내용 해석은 호스트가 공급합니다. 이 렌더 계약(`{lines}` / `{image}`)이 그대로 플러그인 반환 규약의 토대가 되도록 설계돼 있습니다.

> **아직 없는 것**: `PreviewProvider` 추상화, 확장자→공급자 레지스트리, Starlark 런타임, `preview_map` 설정. 전부 로드맵입니다.

## 3. 계획된 플러그인 모델 (로드맵)

> 아래는 ADR-0004에 기록된 설계이며 **아직 구현되지 않았습니다.** 인터페이스는 구현 시 확정됩니다.

### 플러그인 계약

- 위치: `data\plugins\*.star`(포터블 — exe 옆. 파일 추가/수정만으로 배포, 재빌드 불요).
- 각 `.star`는 메타 선언 + 함수를 정의:
  - `ID`(안정 식별자) · `NAME`(표시명) · `EXTS`(지원 확장자 목록)
  - `can_preview(file)` — 선택적 동적 판정
  - `preview(file)` — 미리보기 생성
- **반환 규약**(셋 중 하나): `{"lines": [str]}` · `{"image": path}`(호스트 WIC 위임) · `{"kv": {k:v}}`(정보 표).

### 호스트 API (샌드박스 표면)

이것 외의 I/O는 전면 차단됩니다.

- `file.path` · `file.ext` · `file.size`
- `file.read(n)` — **상한 강제**, bytes 반환
- `file.exif()` — 호스트가 제공하는 순수 러스트 EXIF 파서(TIFF/JPEG APP1)

### 확장자 매핑 · 공급자 선택

1. **기본 매핑** = 플러그인 선언(`EXTS`) → 로드 시 확장자→후보 레지스트리 구성.
2. **설정 오버라이드** = `preview_map`(확장자→플러그인 `ID`).
3. **다중 매치 콤보** = 한 확장자에 후보가 2개 이상이면 도크 미리보기 우상단의 소형 콤보박스로 즉석 전환(선택은 확장자별 기억).
4. **우선순위**: 설정 오버라이드 > 콤보 선택 > 선언 매치 > 내장 폴백.

### 격리 · 예산

- 실행 시간 상한·스텝 제한·오류 시 해당 플러그인만 비활성(도크에 오류 1줄).
- 미리보기 최초 사용 시 **지연 로드** → 상주 RSS 영향 0(B1 예산 보호).

### `.star` 예시 (설계안 — 아직 미동봉)

```python
# data\plugins\exif.star — EXIF 정보 미리보기
ID   = "exif"
NAME = "EXIF 정보"
EXTS = ["jpg", "jpeg", "tiff"]

def can_preview(file):
    return True

def preview(file):
    tags = file.exif()              # 호스트 내장 EXIF 파서(순수 러스트)
    if not tags:
        return {"lines": ["EXIF 없음"]}
    lines = ["{}: {}".format(k, v) for k, v in tags.items()]
    return {"lines": lines}
```

## 4. 구현 로드맵 (슬라이스)

전부 **미착수**입니다(TODO X-2).

| 슬라이스 | 내용 |
| --- | --- |
| **S1 공급자 시임** | 내장 `preview_content`를 `PreviewProvider` 계약 뒤로 리팩터. 내장 = `builtin.text`/`builtin.image` 기본 공급자. 확장자→후보 레지스트리. (러스트만·crate 0) |
| **S2 Starlark 런타임** | `starlark` crate 도입(예산 원장 등재·exe 크기 실측 후 확정), `.star` 로딩·`preview` 호출 배선 |
| **S3 호스트 API + 매핑** | `file` 핸들(read 상한·exif 헬퍼), `preview_map` 오버라이드, 도크 우상단 공급자 콤보(선택 영속) |
| **S4 예시 플러그인** | `exif.star` 동봉 + 개발 가이드 |

> `starlark` crate 도입은 exe 예산(B2 ≤10MB)에 수 MB를 더할 수 있어, 도입 시점에 의존 트리·크기 증가분을 실측한 뒤 승인합니다(DR-8 외부 crate 원장). 초과 시 경량 대안(자체 미니 인터프리터·WASM 재검토)을 결정합니다.
