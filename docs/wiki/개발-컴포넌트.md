# 컴포넌트 개발 (Nx 컨트롤)

`crates/nexa-app/src/ctl/`는 순수 Rust + Win32(windows-rs)로 만든 자체 UI 컴포넌트 세트 **"Nx"**입니다. comctl32 비의존(B3 게이트)·관리 런타임 금지(DR-1)·앱 비결합(향후 독립 라이브러리 지향)을 목표로 설계됐습니다.

> 각 컨트롤 계약의 원천(SSOT)은 소스 모듈 최상단의 doc 주석이며, 이 문서는 그 요약입니다.

---

## 1. 아키텍처 개요

### 공통 기반 — `ctl/base.rs`

모든 Nx 컨트롤이 반복하는 4대 생명주기 패턴을 한곳에 추출한 재사용 계층입니다.

| 헬퍼 | 역할 |
| --- | --- |
| `state<T>(hwnd) -> *mut T` | `GWLP_USERDATA`에서 상태 박스 포인터 읽기(미설치 = 널) |
| `attach_state<T>(hwnd, Box<T>)` | 생성 시 1회, `Box::into_raw`로 소유권 이전 |
| `drop_state<T>(hwnd)` | WM_DESTROY 표준 처리 — USERDATA 비우고 `Box::from_raw`로 회수(Drop 실행) |
| `notify(hwnd, code)` | 부모에 `WM_COMMAND(MAKEWPARAM(id, code))` 재발행(lparam = 컨트롤 HWND) |
| `register_class(once, class, proc)` | `Once` 가드로 창 클래스 1회 등록 |

**RAII 규약**: GDI 자원(HFONT·HBRUSH·GpImage 등)을 보유하는 상태는 `Drop`을 구현해 `drop_state`가 박스를 회수할 때 함께 해제되게 합니다(WM_DESTROY 수동 해제 누락 방지).

### Nx 명명 규약

- 컨트롤 이름 = **Nx 접두어**, 네임스페이스 = **Nexa**.
- Win32 클래스명 = `Nexa.Nx<이름>`, 팝업 창은 `Nexa.Nx<이름>Pop`.
  - 예: `Nexa.NxComboBox` + 팝업 `Nexa.NxComboBoxPop`.
- 모듈 파일명·Rust 심볼은 소문자(`ctl::combobox::create`), 클래스명만 PascalCase.

### 공통 규약

- **판매용 추상화 계약**: 앱 비결합 — 색은 `style::Style` 인자로 주입(하드코딩 금지)·라벨은 복사 소유·i18n/테마 미참조·텍스트는 `WM_SETTEXT/GETTEXT` 위임·통지는 컨트롤 id로 WM_COMMAND 재발행.
- **공통 자동 높이**: `h <= 0`이면 `style::auto_height()`(글꼴 높이 + 상/하 `PAD_Y` 4px) 사용 → 같은 row에 배치해도 세로 크기·시각 중심이 일치. (예외: NxButton·NxSegmented는 "글꼴 + 상/하 2px" 컴팩트 규칙.)
- **AA 렌더 규약**: 곡선·사선 도형은 `DrawCtx`의 AA 프리미티브로만 그린다 — **GDI+ 직접 호출 금지**, 유일 접점은 `gdipctx`(§4). 텍스트는 GDI/DirectWrite 유지.

## 2. 컨트롤 카탈로그

`WM_USER = 0x0400`. 조회/설정 상수는 `0x0400 + n`, 통지 코드는 `WM_COMMAND`의 HIWORD로 전달되는 작은 상수(주로 1)입니다.

| 클래스 | 목적 | 통지/메시지 |
| --- | --- | --- |
| `NxSearchBox` | 내장 ✕ 지우개 검색 입력 | `EN_CHANGE` |
| `NxFontBox` | 글꼴 피커(각 항목 그 글꼴로 렌더) | `FBM_HAS_DROP` · `EN_CHANGE` |
| `NxSegmented` | 세그먼트 라디오(`SegOpts{corner,gap}`) | `SEG_CHANGED` · `SEG_GET/SETSEL` |
| `NxSpin` | 숫자 스피너(min/max 방향 비활성) | `SPIN_GET/SETVAL` · `EN_CHANGE` |
| `NxGroupCard` | 타이틀+본문 카드(자식 통지 투과) | `GC_GETTITLEH` |
| `NxComboBox` | macOS 팝업 버튼(✓ 선택) | `NXCB_CHANGED` · `NXCB_GET/SETSEL` |
| `NxCheckBox` | 라운드 박스 토글(2단/3단) | `NXCHK_CHANGED` · `NXCHK_GET/SETCHECK` |
| `NxTextBox` | 라운드 입력·포커스 accent 링 | `EN_CHANGE` |
| `NxIconButton` | shape 투명 원형 버튼(글리프/이미지/SVG) | `NXIB_CLICK` · `NXIB_GET/SETENABLE` |
| `NxButton` | 푸시 버튼 3상태(Normal/Default/Disabled) | `NXBTN_CLICK` · `NXBTN_SETENABLE/SETDEFAULT` |
| `NxLabel` | 폼 라벨(마우스 투명·열 폭 정렬) | 없음 |
| `NxMenuButton` | `… ⌄` 오버플로 액션 메뉴 | `NXMB_PICK` · `NXMB_GETPICK` |
| `NxGrid` | 그리드(헤더 리사이즈·오버레이 스크롤·행 선택·다중 정렬) | `NXGR_TOGGLE/SELCHANGE/SORT` · `NXGR_GETROW` |
| `NxOrderTree` | 순서 편집 트리(2레벨·드래그·펼침/닫음) | `NXOT_SELCHANGE/TOGGLE/DRAGMOVE` |

> 검증 갤러리 `ctldemo.rs`에서 전 컨트롤의 사용 예시를 볼 수 있습니다. 컨트롤별 상세 문서 세트는 저장소 `docs/ctl/`에 있습니다.

## 3. 스타일 시스템 — `ctl/style.rs`

모든 컨트롤은 색을 하드코딩하지 않고 `Style`을 **생성 인자**로 받습니다.

```rust
pub struct Style {
    pub bg: COLORREF,        // 배경(입력 필드·팝업)
    pub border: COLORREF,    // 1px 테두리
    pub text: COLORREF,      // 본문 글자
    pub text_dim: COLORREF,  // 보조 글자(플레이스홀더·✕)
    pub accent: COLORREF,    // 강조(선택 테두리·활성 세그먼트)
    pub sel_bg: COLORREF,    // 선택/hover 배경
    pub behind: COLORREF,    // 도형 밖 배후색(AA 가장자리 블렌드 기준)
    pub danger: COLORREF,    // 파괴적 액션(삭제 등)
}
```

- **`behind`가 AA의 핵심**: 원형/라운드 컨트롤은 1비트 리전 클립(계단 가장자리의 진범) 대신 이 색으로 모서리를 먼저 칠하고 그 위에 AA 도형을 블렌드합니다. 부모 배경과 같은 값을 넘기면 어떤 배경 위에서도 자연스럽습니다.
- **테마 연동**: `ctl`은 테마를 모릅니다. 호스트(앱)가 현재 테마에 맞는 `Style` 값을 만들어 `create` 시 주입합니다.
- 공용 유틸: `auto_height()`·`text_width()`·`font_height()`·`fill()`·`frame()`.

## 4. 그리기 백엔드 — `ctl/gdipctx.rs`

### 왜 유일 GDI+ 접점인가

GDI 도형(Ellipse/RoundRect/펜)에는 AA가 없고, shape 투명에 쓴 1비트 창 리전 클립이 곡선 계단의 진범이었습니다. 그래서 **"GDI+를 쓰되 DrawCtx 백엔드로만, ctl에서 직접 호출 금지"** 규약을 세웠습니다.

- **이 모듈이 코드베이스의 유일한 GDI+ 접점**입니다. 컨트롤 코드에는 GDI+ 심볼이 0개.
- 컨트롤은 `nexa_gui::DrawCtx` 트레이트의 AA 프리미티브만 호출 — 래스터라이저 교체 시 이 모듈만 바꿉니다.
- `gdiplus.dll` = OS 인박스 → B3 게이트 통과·외부 crate 0 유지.

### 주요 API

- `fill_ellipse` · `fill_round_rect` · `stroke_round_rect` · `polyline` — GDI+ AA(둥근 캡/조인).
- `svg_to_hicon(doc, px, argb)` / `svg_to_image(doc, px, argb)` — SVG를 요청 크기로 즉석 래스터(DPI 임의 크기 선명).
- `decode_png` · `image_size` · `draw_image` · `dispose_image` — 이미지.
- **HDC 혼용 규약**: GDI 텍스트를 그리기 전에 `GdipCtx`를 스코프 블록으로 먼저 해제(Graphics drop 후 GDI `DrawTextW`).

### SVG 파이프라인 — `svg.rs`

외부 crate 0으로 자체 구현한 최소 SVG 서브셋 파서(신뢰된 임베드 자산 전용). 지원: `viewBox`·`stroke-width`·`fill` · `rect(rx)`/`circle`/`line`/`polyline`/`path`(M/L/H/V/C/A[원형]/Z)/`text`. 미지 요소는 건너뜀(오류 격리).

## 5. 새 컨트롤 만드는 법 (튜토리얼)

가장 단순한 `label.rs`를 참조 예시로, 5단계로 최소 컨트롤을 만듭니다.

**단계 0** — `ctl/mod.rs`에 `pub mod myctl;` 추가.

**단계 1** — 상태 구조체 + 정적 가드 + 클래스명:
```rust
struct MyState { font: HFONT, style: Style }
static REGISTER: std::sync::Once = std::sync::Once::new();
const CLASS: PCWSTR = w!("Nexa.NxMyCtl");   // Nx 명명 규약
```

**단계 2** — 통지 상수(호스트 계약):
```rust
pub const NXMY_CLICK: u32 = 1;              // WM_COMMAND HIWORD 통지
pub const NXMY_GETVAL: u32 = 0x0400 + 130;  // WM_USER+n 조회/설정
```

**단계 3** — `create`: 클래스 등록 → 창 생성 → 상태 박싱:
```rust
pub unsafe fn create(parent: HWND, x: i32, y: i32, w: i32, h: i32,
                     id: u32, font: HFONT, style: Style) -> HWND {
    super::base::register_class(&REGISTER, CLASS, Some(proc));
    let h = if h <= 0 { super::style::auto_height(parent, font) } else { h };
    let hwnd = CreateWindowExW(/* … WS_CHILD|WS_VISIBLE, id를 HMENU로 … */);
    super::base::attach_state(hwnd, Box::new(MyState { font, style }));
    SendMessageW(hwnd, WM_SETFONT, Some(WPARAM(font.0 as usize)), Some(LPARAM(1)));
    hwnd
}
```

**단계 4** — `proc`(wndproc): WM_ 라우팅 + paint + notify:
```rust
unsafe extern "system" fn proc(hwnd, msg, wp, lp) -> LRESULT {
    match msg {
        WM_DESTROY => { super::base::drop_state::<MyState>(hwnd); LRESULT(0) }
        WM_LBUTTONDOWN => { super::base::notify(hwnd, NXMY_CLICK); LRESULT(0) }
        WM_PAINT => {
            // fill(dc, &rc, st.style.behind);         // 배후색 먼저(AA 블렌드 기준)
            // { let mut g = GdipCtx::new(dc); g.fill_round_rect(...); }  // GDI 텍스트 전 해제
            // DrawTextW(...)                            // 텍스트는 GDI로 직접
            LRESULT(0)
        }
        _ => DefWindowProcW(hwnd, msg, wp, lp),
    }
}
```

### 체크리스트

- [ ] 클래스명 `Nexa.Nx*` · `mod.rs` 등록 · 모듈 doc에 계약 명시.
- [ ] `h<=0` 공통 자동 높이.
- [ ] 색은 전부 `Style`에서 — 하드코딩 0.
- [ ] AA 도형은 `GdipCtx`만 · 텍스트는 GDI · Graphics는 GDI 텍스트 전에 해제.
- [ ] GDI 자원 보유 시 `impl Drop` · WM_DESTROY는 `drop_state`.
- [ ] **빈 문자열 셀은 그리지 않는다**(빈 `Vec<u16>` → `DrawTextW` = user32 AV).

## 6. 통지 규약

### 기본형 — WM_COMMAND 재발행

호스트(부모)는 자식 컨트롤 이벤트를 표준 `WM_COMMAND`로 받습니다. `base::notify(hwnd, code)`가:

```
SendMessageW(parent, WM_COMMAND, MAKEWPARAM(id, code), hwnd)
```

즉 **WPARAM LOWORD = 컨트롤 id**, **HIWORD = 통지 코드**, **LPARAM = 컨트롤 HWND**. 클래식 Win32 컨트롤 관례와 동일해 기존 배선을 재사용할 수 있습니다.

```rust
WM_COMMAND => {
    let id   = (wparam.0 & 0xFFFF) as u32;
    let code = ((wparam.0 >> 16) & 0xFFFF) as u32;   // NXBTN_CLICK, NXCB_CHANGED …
}
```

### 특이 규약

- **SETxxx 계열(WM_USER+n)은 통지를 재발행하지 않습니다**(무한 루프 방지).
- **NxGroupCard**는 자식이 아니라 투과자 — 얹은 자식의 통지를 카드 부모로 그대로 전달.
- **NxGrid**: 정렬은 컨트롤이 통지만 하고 **비교는 호스트**가 수행(`set_rows`로 재정렬).
- **팝업 owner 함정**: 콤보/메뉴버튼의 `WS_POPUP` 드롭다운은 자식 컨트롤을 owner로 생성하면 시스템이 owner를 최상위로 승격시켜 `GetParent`가 엉뚱한 창을 반환합니다. → owner HWND를 **팝업 자신의 USERDATA에 저장**해 복원합니다.
